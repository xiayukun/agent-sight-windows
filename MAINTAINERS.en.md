# Maintainers

[中文](MAINTAINERS.md) | English

## Current Maintenance Policy

- The project is still research/MVP software.
- Before public release, repository name, release assets, evidence cleanup, license, and screenshots require operator approval.
- Engineering stages should run tests and receive an independent review conclusion; by default, subagent reviews are no longer persisted as local `docs/reviews` files.

## Pre-Release Maintainer Checks

- README / README.en.md first screen is aligned;
- release notes are updated in Chinese and English;
- `dist/` artifacts and SHA256 checksums come from GitHub Actions;
- evidence / runs / local cache files are not in the published source;
- Run Key / Startup points to the current packaged Supervisor.
